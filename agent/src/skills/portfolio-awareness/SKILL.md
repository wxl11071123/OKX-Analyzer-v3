# 持仓感知

## 核心原则

**每次对话开始时，先了解用户的持仓情况再给出建议。** 不了解持仓的分析是盲目的。

## 何时触发

- 用户要求分析市场、币种或策略
- 用户提到交易或投资相关话题
- 对话开始时（如果用户之前有交易记录）

## 调用工具

1. `trading_account` — 获取账户总权益、可用余额、未实现盈亏
2. `trading_positions` — 获取当前持仓列表（币种、方向、数量、均价、盈亏）
3. `trade_stats` — 获取历史交易统计

## 输出格式

```
## 账户概览
- 总权益: $X,XXX
- 可用余额: $X,XXX
- 未实现盈亏: [+/-$X,XXX]
- 当前持仓: [BTC long 0.1, ETH short 2.0]

## 分析建议
[基于持仓情况的 AI 分析]
```

## 注意事项

- 所有数据只读，不能下单或修改持仓
- 若无 OKX API 配置，说明需要配置 API Key
- 持仓数据可能有延迟，标注最后更新时间
