# 交易日志分析

## 核心原则

**了解用户怎么交易，才能给出有用的建议。** 历史交易数据是最好的老师。

## 何时触发

- 用户要求分析交易行为
- 讨论策略或给出建议时
- 每次对话开始时检查是否有新交易记录

## 调用工具

1. `query_trade_log` — 查询交易记录（可按币种、类型过滤）
2. `trade_stats` — 获取交易统计（胜率、总盈亏、平均纪律评分）
3. `sync_trade_log` — 从 OKX 同步最新成交记录（如果需要）

## 分析维度

| 维度 | 指标 |
|------|------|
| 盈利能力 | 总盈亏、胜率、盈亏比 |
| 风险管理 | 最大单笔亏损、平均持仓时间 |
| 纪律性 | 平均纪律评分、是否有无止损的交易 |
| 交易偏好 | 最常交易的币种、偏好的方向（多/空） |
| 行为偏差 | 过度交易、止损犹豫、盈利过早止盈 |

## 输出格式

```
## 交易日志摘要
- 总交易: X 笔
- 胜率: XX%
- 总盈亏: [+/-$X,XXX]
- 平均纪律评分: X.X/10

## 关键发现
1. [发现1]
2. [发现2]

## 改进建议
1. [建议1]
2. [建议2]
```

## 注意事项

- 如果用户还没有同步交易记录，提醒先配置 OKX API Key 并调用 sync_trade_log
- 用户手动添加的备注（note）和纪律评分（discipline_score）要特别关注
- AI 不能修改交易记录，只能读取和分析
